507.cactuBSSN_r

Based on the Cactus Computational Framework, this benchmark uses the EinsteinToolkit to solve the Einstein equations in vacuum. The numerical kernel of this benchmark, McLachlan, is automatically generated from a high level set of partial differential equations with the Kranc code generation package. As a production level code, McLachlan has been successfully used to model blackholes, gravitational waves. When coupled to other general relativistic (magneto)hydrodynamic codes, it can also be used to model white dwarfs, neutron stars, etc. In this benchmark, we model a vacuum flat space-time.

Official description is available at https://www.spec.org/cpu2017/Docs/benchmarks/507.cactuBSSN_r.html.

DISCLAIMER: All the SPEC results are "non-reportable". The measurements were taken in order to compare different compilers, not different hardware, please do not attempt to make any comparisons of the different machines used. Moreover, we have broken various SPEC benchmarking rules, we have even patched some benchmark sources to be able to compile them with -fstrict-aliasing, build with -fno-common, and so forth. And the data may be just wrong, the scripts generating all the fancy graphs and tables have been known to contain bugs in the past. On the other hand, this is the data that we use to evaluate where GCC stands and we believe that overall this comparison of compilers is a useful one.
